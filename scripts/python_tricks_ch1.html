🔒 Assert — Утверждения

Инструмент для проверки инвариантов, а не обработки ошибок

💡
Инвариант — условие, которое всегда должно быть истинным. Если оно нарушено — это баг в программе, а не ошибка пользователя.
❌ Неправильно
def delete_user(user_id):
    # Нельзя — assert можно отключить!
    assert user_id > 0, "ID > 0"
    db.delete(user_id)
✅ Правильно
def delete_user(user_id):
    # Для внешних данных — ValueError
    if user_id <= 0:
        raise ValueError("ID > 0")
    db.delete(user_id)

Пример — проверка скидки

✅ Правильное использование 
def apply_discount(price, discount):
    # Скидка — внутренний параметр.
    # Значение вне [0, 1] — это баг в коде, не ввод пользователя.
    assert 0 <= discount <= 1, f"Скидка должна быть от 0 до 1, получено: {discount}"
    return price * (1 - discount)

print(apply_discount(1000, 0.2))   # → 800.0
print(apply_discount(500, 0.5))    # → 250.0
print(apply_discount(100, 1.5))    # → AssertionError: Скидка должна быть от 0 до 1
Вывод
800.0
250.0
AssertionError: Скидка должна быть от 0 до 1, получено: 1.5
⚠️
Главная ловушка: Python позволяет отключить все assert одним флагом:
python -O script.py
Никогда не используй assert для валидации пользовательского ввода, данных из БД или внешних API.

Assert с кортежем — классический баг

⚠️ Ловушка 
# ВСЕГДА True! Кортеж непустой → True
assert(1 == 2, "Это должно упасть")   # ← НЕ упадёт!

# Правильно — без скобок:
assert 1 == 2, "Это упадёт"           # ← AssertionError
Вывод
# Первая строка не бросает исключение (кортеж всегда True)
# Вторая строка: AssertionError: Это упадёт

✏️ Висящие запятые

Маленькое соглашение, которое предотвращает незаметные баги

❌ Баг
names = ['Иван',
         'Мария'   # ← нет запятой!
         'Сергей']

print(names)
# ['Иван', 'МарияСергей']
# Python склеил строки!
✅ Правильно
names = [
    'Иван',
    'Мария',
    'Сергей',    # ← висящая запятая
]

print(names)
# ['Иван', 'Мария', 'Сергей']
💡
Python автоматически склеивает строковые литералы стоящие рядом: 'Мария' 'Сергей' → 'МарияСергей'. Это фича для длинных строк, но ловушка в списках.

Работает везде: dict, tuple, аргументы функций

Хорошая практика 
# Словарь
user = {
    'name': 'Иван',
    'email': 'ivan@example.com',
    'role': 'admin',      # ← можно добавить новый ключ без правки предыдущей строки
}

# Аргументы функции
def create_user(
    name: str,
    email: str,
    role: str = 'user',   # ← висящая запятая
) -> dict:
    return {'name': name, 'email': email, 'role': role}

# Вызов функции
result = create_user(
    name='Иван',
    email='ivan@example.com',
    role='admin',         # ← можно добавить kwarg в конец
)

Git diff без лишних изменений

Бонус 
# Без висящей запятой — при добавлении элемента меняются 2 строки:
# -    'Сергей'
# +    'Сергей',
# +    'Алексей'

# С висящей запятой — меняется только 1 строка:
# +    'Алексей',

📦 Context Managers

Паттерн «открыть → использовать → закрыть» гарантированно

❌ Утечка ресурса
f = open('data.txt', 'r')
data = f.read()  # исключение здесь?
f.close()        # тогда не выполнится!
✅ Гарантированное закрытие
with open('data.txt', 'r') as f:
    data = f.read()
# f.close() — вызовется всегда

Как работает изнутри — магические методы

Механика 
class ManagedFile:
    def __init__(self, name: str):
        self.name = name

    def __enter__(self):
        """Вызывается при входе в блок with. Возвращает объект после 'as'."""
        print(f"Открываю {self.name}")
        self.file = open(self.name, 'r')
        return self.file

    def __exit__(self, exc_type, exc_val, exc_tb):
        """Вызывается при выходе из with — всегда, даже при исключении."""
        print(f"Закрываю {self.name}")
        self.file.close()
        return False   # False = не подавлять исключения

# Использование:
with ManagedFile('data.txt') as f:
    content = f.read()
Вывод
Открываю data.txt
Закрываю data.txt

Проще — декоратор @contextmanager

Рекомендуется 
from contextlib import contextmanager
import time

@contextmanager
def timer(label: str):
    """Замеряет время выполнения блока."""
    start = time.monotonic()
    print(f"⏱ {label} — старт")
    try:
        yield          # всё до yield = __enter__
    finally:
        elapsed = time.monotonic() - start
        print(f"⏱ {label} — {elapsed:.3f} сек")  # finally = __exit__

# Использование:
with timer("Загрузка данных"):
    time.sleep(0.5)
    data = [i ** 2 for i in range(100000)]
Вывод
⏱ Загрузка данных — старт
⏱ Загрузка данных — 0.512 сек

Реальный пример — подключение к БД

Практика 
from contextlib import contextmanager

@contextmanager
def db_transaction(connection):
    """Автоматический commit или rollback."""
    try:
        yield connection
        connection.commit()
        print("✅ Транзакция зафиксирована")
    except Exception as e:
        connection.rollback()
        print(f"❌ Откат транзакции: {e}")
        raise

with db_transaction(conn) as db:
    db.execute("INSERT INTO users VALUES (?)", (1, "Иван"))

_ Подчёркивания и дандеры

Соглашения об именовании — 5 видов, каждый со своим смыслом

Вид Название Смысл
_var Внутренний «Не трогай снаружи». Не экспортируется через import *
__var Приватный (mangling) Python переименует в _ClassName__var. Реально скрыт.
__var__ Дандер (dunder) Магические методы Python. Не создавай свои!
var_ Без конфликта Избегаем совпадения с ключевым словом: class_, type_
_ Одноразовый Значение не нужно. Также — последнее значение в интерпретаторе.

_var — одно подчёркивание

Соглашение 
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner      # публичный — используй смело
        self._balance = balance  # внутренний — лучше не трогать напрямую

    def deposit(self, amount):
        self._balance += amount

    def get_balance(self):
        return self._balance

account = BankAccount("Иван", 1000)
print(account.owner)      # → 'Иван'  ✓
print(account._balance)   # → 1000   (работает, но нарушает соглашение!)
Вывод
Иван
1000

__var — name mangling

Сильная защита 
class SecureVault:
    def __init__(self, secret):
        self.__secret = secret    # Python переименует в _SecureVault__secret

    def reveal(self):
        return self.__secret      # внутри класса работает как обычно

vault = SecureVault("abc123")
print(vault.reveal())            # → 'abc123'  ✓

# Прямой доступ снаружи:
# vault.__secret                 → AttributeError!
# vault._SecureVault__secret     → 'abc123' (обходной путь)

# Зачем это нужно? Защита при наследовании:
class EvilVault(SecureVault):
    def __init__(self):
        self.__secret = "взломан"   # → _EvilVault__secret (другое имя!)
        super().__init__("original")

ev = EvilVault()
print(ev.reveal())  # → 'original' (не перезаписано!)
Вывод
abc123
original

_ — одноразовая переменная

Чистый код 
# Цикл без счётчика
for _ in range(5):
    print("Привет!")    # выведет 5 раз

# Распаковка — нужны только первый и последний
first, *_, last = [1, 2, 3, 4, 5]
print(first, last)      # → 1 5

# Координаты — нужны только x и z
x, _, z = get_coordinates()

# В интерпретаторе _ = предыдущий результат
# >>> 2 + 2
# 4
# >>> _ * 3
# 12
Вывод
Привет!  (× 5)
1 5

💬 Форматирование строк

Четыре способа — когда какой использовать

Все четыре способа

name = "Боб"
age  = 30

# 1. %-форматирование (старый C-стиль, Python 2)
result = "Привет %s, тебе %d лет" % (name, age)

# 2. str.format() (Python 2.7+)
result = "Привет {}, тебе {} лет".format(name, age)
result = "Привет {name}, тебе {age} лет".format(name=name, age=age)

# 3. f-строки (Python 3.6+) ← ИСПОЛЬЗУЙ ЭТО
result = f"Привет {name}, тебе {age} лет"
result = f"Через 10 лет: {age + 10}"   # можно выражения!

# 4. Template (для пользовательских шаблонов)
from string import Template
t = Template("Привет $name, тебе $age лет")
result = t.substitute(name=name, age=age)
result = t.safe_substitute(name=name)   # не бросает при отсутствии переменной

Суперсилы f-строк

Python 3.6+ 
price = 1_234_567.89
pi    = 3.14159265
name  = "Python"
x     = 42

# Форматирование чисел
print(f"{price:,.2f}")      # → 1,234,567.89
print(f"{pi:.4f}")          # → 3.1416
print(f"{42:08b}")          # → 00101010  (двоичное, 8 знаков)
print(f"{0.75:.1%}")        # → 75.0%

# Выравнивание текста
print(f"|{'влево':<12}|")   # → |влево       |
print(f"|{'центр':^12}|")   # → |   центр    |
print(f"|{'вправо':>12}|")  # → |      вправо|

# Отладка (Python 3.8+) — самое удобное!
print(f"{x = }")            # → x = 42
print(f"{price = :,.0f}")   # → price = 1,234,568
Вывод
1,234,567.89
3.1416
00101010
75.0%
|влево       |
|   центр    |
|      вправо|
x = 42
price = 1,234,568
⚠️
f-строки выполняют код! Никогда не используй f-строки с пользовательским вводом:
template = input() → f"{template}" — это уязвимость.
Для шаблонов от пользователя используй Template.safe_substitute()

Когда что использовать

Правило Бейдера 
# f-строки — везде (по умолчанию)
message = f"Пользователь {user.name} вошёл в {datetime.now():%H:%M}"

# Template — если шаблон приходит снаружи (файл, БД, пользователь)
email_body = Template(open('templates/email.txt').read())
sent = email_body.safe_substitute(name=user.name, order_id=order.id)

# .format() — если нужна совместимость с Python < 3.6
# % — только в легаси-коде, новый код не пиши так
